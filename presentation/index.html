Why We Built This

The Problem

😤

No Visibility Into Server Availability

Engineers didn't know which lab servers were free, who was using them, or for how long — leading to constant Slack messages and email chains.

⚡

Double-Booking & Conflicts

Multiple teams would SSH into the same server simultaneously, corrupting test environments and wasting hours of engineer time.

🔒

Resource Hoarding

Servers would sit idle for weeks while booked indefinitely with no accountability, blocking other teams from running critical tests.

📊

Zero Utilization Data

No way to track server utilization, team usage patterns, or capacity planning — infrastructure decisions were made on guesswork.

🔐 Authentication Flow

JWT tokens in localStorage → Every request validates JWT and checks session exists in DB → server-side logout supported by deleting session row.

📡 API Design

RESTful routes under /api/. Role-based: public routes (login, register), user routes, admin-only routes. Rate-limited + CORS-protected.

🤖 Agent Layer

Cron-driven monitoring cycle every 4 hours. Checks expiring bookings, syncs server status, generates AI summaries via Azure OpenAI Responses API.

📦 Docker Compose

Three containers: database (Postgres), backend (Node.js), frontend (Nginx). Named volume ensures data persistence across restarts.

AI Integration

LabOps Agent — AI Assistant

🤖 What Is It?

An always-on AI assistant powered by Azure OpenAI (GPT-4o) embedded directly in the dashboard. Engineers chat with it to get instant answers about server availability, bookings, and operations.

⚙️ How It Works

Connects to Azure OpenAI Responses API using api-key authentication. Formats messages with instructions (system) and input (conversation) fields per the Responses API spec.

🕐 Scheduled Cycles

Runs every 4 hours via node-cron: checks expiring bookings, sends renewal reminders, marks overdue bookings as completed, logs AI insights to the agent_logs table.

Example questions engineers can ask:

"What servers are available right now?"
"Which bookings are expiring this week?"
"Show me utilization for the GPU cluster"
"Who is using XEON-SPR-001?"
"Summarise this week's lab activity"

🔌 Endpoint Used

https://kkopenailearn.openai.azure.com/openai/responses?api-version=2025-04-01-preview

Security & Compliance

Security Controls & Future Compliance

Implemented Today

✓

JWT + Server-Side Sessions

Token + DB session validation on every request. Logout invalidates token server-side.

✓

Password Hashing (bcrypt)

bcryptjs with cost factor 10. Plaintext passwords never stored.

✓

Helmet.js Security Headers

CSP, HSTS, X-Frame-Options, X-Content-Type-Options set on all responses.

✓

Rate Limiting + CORS

100 req/15min on all /api/* routes. Strict origin allowlist enforced.

✓

Input Validation + ORM Queries

Prisma parameterised queries prevent SQL injection. Request body size limited.
